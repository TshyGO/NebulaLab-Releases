## ✨ 为什么选择 Nebula Graph？
Nebula Graph 专为需要处理 **成组相似数据**（而非单一离散样本）的研究者、工程师和科学家设计，我们首次将“数据预处理（Data Preparation）”视为同绘图一样重要的一等公民环境。
从原始数据到高质量图文报告的转化，只需丝滑的一瞬：  
`📥 导入` &nbsp;➔&nbsp; `📂 分组` &nbsp;➔&nbsp; `🧹 清洗` &nbsp;➔&nbsp; `⚙️ 批量处理` &nbsp;➔&nbsp; `🌿 分支处理` &nbsp;➔&nbsp; `📊 可视化探查` &nbsp;➔&nbsp; `📤 一键导出`

## 🚀 快速起步

1. 📥 **下载客户端**：前往 [Releases 下载页](https://github.com/TshyGO/NebulaGraph-Releases/releases/latest) 获取安全签名安装包。
2. 💻 **完成安装**：部署于任意常规 Windows 工作站设备。
3. 📂 **拖拽入库**：载入各种仪器产出的无序格式原始谱图或数据簿（支持 TXT、CSV 及 Excel 工作簿）。
4. ⚙️ **即刻清洗**：自动侦测格式壁垒，直观进行表格微操。
5. 📊 **分析验证**：用你习惯的 Workflow 套件进行快速预分析。
6. 📤 **生成绘图级资产**：彻底丢掉格式苦恼，直接投递给 Origin (*.opju / *.oth) 等第三方专业绘图软件。

## 🙋 常见疑惑答疑 (FAQ)

**Nebula Graph 是纯粹的开源软件吗？**  
目前面向大家开放的库 `NebulaGraph-Releases` 仅用作二进制归档和客户端的无头分发站。软件主应用项目现挂载于内部核心仓库中。

**主页上显示的 Source code (zip) 等包是做什么的？**  
那是 GitHub Release 页原生机制生成的源码说明书镜像，完全不包含实际应用代码。

**现在大概是什么成熟阶段？**  
当前项目处于高速、高频迭代的 **Alpha 测试阶段**。核心流逻辑基本实现稳定闭环，UI 美化与多图同步逻辑在快速打磨中！
